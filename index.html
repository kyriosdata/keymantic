Entendendo o keymantic...

Definições

Palavra-chave

Uma palavra-chave é uma sequência de caracteres. Exemplos: heart, a sigla IMC, o valor 1200 ou ainda 80mm/HG.

Observe que uma palavra-chave pode conter um ou mais espaços em branco e, nesse caso, a palavra-chave deve ser obrigatoriamente delimitada por aspas duplas. Por exemplo, o nome próprio "João Carlos", ou mesmo "80 mm/HG".

Conjunto de arquétipos

Um conjunto de arquétipos $A = \{a_1, a_2, \ldots, a_n\}$ reúne zero ou mais arquétipos $a_i$, onde $0\lt i \leq n$ e $|A|=n$.

Um arquétipo pode, ao longo do tempo, ser alterado, e cada versão de um arquétipo é tratada como um arquétipo distinto.

Consulta baseada em palavras-chave

Uma consulta $q$ é uma sequência de palavras-chave, ou seja, $q = (k_1,k_2,\ldots,k_j)$ onde $j$ é o total de palavras-chave da consulta. Adicionalmente a palavra-chave $k_m$ ocorre imediatamente após $k_{m-1}$ e imediatamente antes de $k_{m+1}$ para $1\lt m\lt j$.

O que é o Keymantic?

O que Keymantic é uma ferramenta que gera uma ou mais consultas para bancos de dados relacionais a partir de uma análise semantica sobre um conjunto de uma ou mais palavras chave. A ferramenta consegue cumprir esta tarefa a partir de uma série de etapas, como o mapeamente das palavras chaves para termos de esquema ou de valor do banco de dados, a partir da execução de algoritmos que geram pesos intrísecos para estas plavras-chave; seleção dos melhores mapeamentos, baseada na contextualização dos termos e pesos encontrados, geração de configurações e interpretações para estas palavras-chave.

Por quê o Keymantic é importante?

As buscas com palavras-chave são a principal forma de pesquisa por informações na web. Contudo, atualmente apenas uma pequena fatia dos das informações acessadas na web são pesquisaveis através dos métodos de pesquisa tradicionais, uma vez que a maior parte dos dados da web estão armazenados em bancos de dados relacionais e estes oferecem suporte limitado para consultas com palavras-chave. Neste cenário, o Keymantic se apresenta como uma opção e forte referência para o acesso a esses dados, que compõe o que chamamos de "Web invisivel", primeiro porque ele não exige do usuário, um conhecimento prévio da estrutura do banco de dados e, segundo, porque não exige dominio de nenhuma linguagem estruturada como SQL.

Os passos do keymantic

O processo de transformação de uma cadeia de caracteres que compoe o conjunto de palavras-chave da consulta até a geração efetiva de um conjunto de uma ou mais palavras consultas para bancos de dados relacionais, que é realizado pelo Keymantic acontece através de cinco passos, descritos abaixo. Ao longo de processo uma estrutura de dados especial, chamada matriz de pesos é utilizada constantemente. Essa é uma matriz bidimensional com uma linha para cada palavra-chave na consulta e uma coluna para cada termo do banco de dados. Os termos do banco de dados podem ser termos de valor ou de esquema. Os termos de esquema referem-se as operações, nomes de tabelas ou nomes de colunas nas tabelas do banco de dados, enquanto que os termos de valor referem-se a possiveis valores que podem estar presentes nas tuplas do banco de dados.

Pré-processamento

Nesta etapa é feita a "limpeza" da consulta, ou seja, remover dela as palavras chaves que não possuem nenhum significado real na base de dados. Outra tarefa importante dessa fase é a segmentação da consulta de forma que palavras chaves que precisem estar juntas para manter seu significado sejam mapeadas como um unico termo. Esta fase é dividida em duas etapas: Verificação de função agregada/valores compostos/ordenação e Expansão da consulta. Esta é um fase de pré-processamento, ou seja, não é considerada um dos cinco passos do keymantic, mas sim uma fase de preparação da palavra da chave para facilitar o trabalho realizados nos passos verdadeiros.

Cálculo dos pesos intrisecos de valor e de esquema

Neste primeiro passo os termos existentes na pesquisa são mapeados através da matriz de pesos para indicar se aquele termo será interpretado como um termo de esquema ou de valor. O calculo desses pesos é feito a partir do uso de algoritmos especificos para esta tarefa.

Seleção dos melhores mapeamentos para o termos de esquema

Neste passo, a ferramenta keymantic utiliza uma versão modificada do algoritmo Hungarian para gerar uma série de mapeamentos entre as palavras-chave e os termos de esquema da submatriz SW (a submatriz SW é o conjunto de colunas da matriz de pesos cujas colunas representam termos de esquema do banco de dados, por exemplo, o nome de um atributo de tabela). Para gerar um mapeamento, é identificado o maior peso de cada linha da tabela, a intesecções de cada linha (uma palavra-chave) com a coluna (um termo de esquema) onde está o valor máximo forma uma mapeamento entre palavras-chave e termos do banco de dados. Esse processo segue até que a quantidade de mapeamentos desejada seja gerada e sempre que um mapeamento é gerado os valores máximos utilizado são removidos, para permitir a geração de um novo mapeamentos entre palavras chave e mapeamentos diferentes. O primeiro mapeamento é considerado melhor que o segundo, e cada mapeamento subsequente é considerado melhor que o anterior.
